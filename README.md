# Objetivo
```El objetivo es transformar el informe de Power BI del Laboratorio 2 en un producto de datos final y profesional, manteniendo la restricción de una sola página. Esto se logra aplicando la integración de una narrativa de datos (Storytelling) que guíe al usuario a la conclusión de negocio.```

## Base del Proyecto

Debe partir del archivo .pbix del Laboratorio 2 o puede cambiar los dataset.

# I. Fase de Data Storytelling y Diseño (El "Contar la Historia")

El informe debe dejar de ser una colección de gráficos y pasar a ser una herramienta de decisión con un flujo narrativo claro, todo en un único lienzo.

## 1.1. Estructura Narrativa Obligatoria

El diseño del lienzo debe seguir una jerarquía visual clara, dividida implícitamente en tres zonas para contar la historia de datos:

Zona Superior (Visión General - El 'QUÉ'):
Debe contener los KPIs más relevantes en Tarjetas de KPI grandes y claras.
Propósito: Responder rápidamente a la pregunta: "¿Cómo estamos?"
 

Zona Central (Análisis Detallado - El 'POR QUÉ'):
Mantener y mejorar los visuales del Laboratorio 2 (Mapa obligatorio, tendencia, distribución).
Propósito: Permitir al usuario la exploración (con los segmentadores) y el diagnóstico: "¿Dónde o por qué está sucediendo esto?"
 

Zona de Conclusión (Mensaje Clave - El 'ENTONCES QUÉ'):
Reservar un espacio destacado (ej. lateral o inferior) para el mensaje final.
Contenido: Utilizar un Cuadro de Texto o el visual de Smart Narrative para escribir 2-3 frases de conclusión y una Recomendación de Negocio clara, basada en los hallazgos de los datos.

## 1.2. Mejoras de Diseño y Comunicación

Título Principal: El dashboard debe tener un título claro que defina el objetivo.
Anclaje Visual: Usar formas, colores de fondo o iconos para dirigir la vista del usuario a la Zona de Conclusión.
Segmentadores: El Panel de Segmentadores debe ser eficiente y no dominar el lienzo (usar de Bookmarks/Marcadores para esconderlo/mostrarlo).
